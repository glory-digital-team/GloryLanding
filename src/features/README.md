# features — слой пользовательских сценариев

Здесь живут **фичи** — самостоятельные действия пользователя, несущие бизнес-ценность
(например: `auth-by-email`, `subscribe-form`, `toggle-theme`).

## Правила

- Может импортировать **только** из `entities` и `shared`.
- Не может импортировать из `widgets`, `views`, `app` и из других `features`.
- Каждая фича — это slice со своим публичным API (`index.ts`).

## Структура среза

```
features/subscribe-form/
├── ui/            # компоненты фичи
├── model/         # состояние, типы, логика (хуки, редьюсеры)
├── api/           # запросы к бэкенду (по необходимости)
└── index.ts       # публичный API: наружу торчит только то, что здесь экспортировано
```
